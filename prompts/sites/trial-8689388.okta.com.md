## Okta Admin Console — Site Context

**Base URL:** https://trial-8689388-admin.okta.com
**MyApps URL:** https://trial-8689388.okta.com (different subdomain — admin console is on -admin subdomain)
**Framework:** MUI (Material UI) — standard HTML/CSS with MuiButtonBase-root, MuiButton-root classes. No Vuetify.

### Authentication

**Auth type: manual_required** — OAuth2 PKCE redirect. The extension CANNOT automate Okta login.

- If the session has expired, the user is redirected to `/oauth2/v1/authorize?...`
- Tell the user: "Your Okta session has expired. Please log in, and I'll continue when you're back."
- Authenticated landing page: `/app/UserHome?session_hint=AUTHENTICATED` (on main domain) or `/admin/users` (on -admin subdomain)
- Do NOT attempt to automate login — Okta may require MFA. Wait for the user.

### DOM Patterns (MUI-based)

Okta Admin Console (trial) uses MUI (Material UI). Key patterns:
- **Read selector:** default `body` — the standard read captures buttons, links, inputs, tables
- **Search:** `textarea[placeholder*='Search for users']` — confirmed working
- **User table:** `table` with headers `[Person & username, Primary email, Status]` — rowCount=8 as of April 2026 (includes Yeshie Test user)
- **Add person:** `click_text` with text "Add person" — confirmed visible on `/admin/users`
- **User profile links:** `a[href*='/admin/user/profile/view/']` — one per user in table
- **Form inputs:** `input[name='profile.firstName']`, `input[name='profile.lastName']`, `input[name='profile.login']`, `input[name='profile.email']`
- **Submit button:** `input[type='submit']`
- **Error selectors:** `.o-form-input-error`, `.inlineError`, `[data-se='error-summary']`
- **Navigation tabs:** hash-based links (e.g., `#tab-apps`, `#tab-groups`, `#tab-account`)
- **More Actions dropdown:** `click_text("More Actions")` — contains lifecycle actions (Deactivate, Suspend, etc.)
- **Confirmation buttons:** `click_text("OK")` for deactivation confirm (not "Deactivate")
- **Groups list:** `table` with columns [Group name, People, Applications]
- **Add group:** `click_text` with text "Add group"
- **Applications list:** `table` with columns [App name, Status, Client ID]
- **Create App Integration:** `click_text` with text "Create App Integration"
- **Browse App Catalog:** link to `/admin/apps/add-app`
- **Assign Users to App:** link to `/admin/app/bulk-assign`
- **Status filter:** readonly textbox showing current filter value
- **Advanced search:** `click_text` with text "Advanced search ▾"
